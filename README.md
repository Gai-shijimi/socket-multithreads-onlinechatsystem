TCP通信とUDP通信を使用したオンラインチャットシステム

# 概要
  TCP通信でルーム名・ユーザー名・部屋を作る/参加するモード等（今回はプログラム上にすでに記載した）を選択すると、サーバー側でクライアントのトークンを管理するようになり、トークンをクライアントに渡す。そして受け取ったトークンを使用してクライアントは指定した部屋で他のユーザーと通信できる仕組み。

## 苦労した点
サーバー側において、スレッドを使用して受信と送信を並列に処理させる構造の作成とクライアントをどう管理するかという点。
TCP通信とUDP通信の２つを同時に用いる必要があった点。（学習当時のTCPとUDPの理解の浅さのため）

## 改善したいこと
クライアントがチャットを正常に終了できるようサーバー側の処理をどのようにしていくかということ。
様々なエラーの対処したプログラムにすること。
